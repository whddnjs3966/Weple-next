# 네이버 로그인 검수 가이드 및 제출 프로세스

이 문서는 네이버 개발자센터에서 [네이버 로그인 사전 검수]를 통과하기 위한 가이드입니다.

## 1. 사전 검수 신청 절차
1. **네이버 개발자센터 접속**: [Application > 내 애플리케이션] 메뉴로 이동합니다.
2. **애플리케이션 선택**: 검수를 받을 애플리케이션을 클릭합니다.
3. **검수 상태 탭 이동**: 좌측 메뉴에서 `네이버 로그인 검수 상태`를 클릭합니다.
4. **자료 첨부 및 요청**: 우측 측면 또는 하단 영역에서 아래 준비된 '검수용 증빙 자료(PDF 등)'를 첨부하고 `검수 요청` 버튼을 누릅니다.
5. **결과 대기**: 영업일 기준 2~3일 내에 등록된 이메일로 검수 결과(승인 또는 반려)가 도착합니다.

## 2. 네이버 로그인 버튼 가이드라인 확인
네이버 로그인의 버튼은 네이버에서 규정하는 디자인 가이드를 반드시 준수해야 합니다. (버튼 비율, 색상, 로고 형태 등)
- 임의로 수정한 버튼이나 '네이버를 연상시키는 다른 로고'를 사용하면 반려됩니다.

## 3. 검수에 필요한 제출 서류 (스크린샷)
검수를 위해 서비스 화면의 캡처본들을 하나의 문서(PDF, Word, PPT 등)로 모아서 제출하는 것이 좋습니다. 이를 본 프로젝트의 `naver_login_review_template.md` 파일로 만들어두었으니 활용해 보세요.

필수 캡처 화면:
- **로그인 버튼 노출 화면**: 웹사이트의 로그인 화면
- **로그인 진행 과정**: 로그인 버튼 누른 뒤 나오는 동의 화면 및 아이디/비밀번호 입력 화면
- **회원가입 완료까지의 화면**: 동의 후 사이트로 돌아와서 회원가입이 마무리되는 전체 플로우 화면
- **추가 정보 활용 증명 (중요)**: 만약 이메일, 성별, 나이, 전화번호 등 추가 정보를 수집한다면, **해당 정보가 서비스 내(예: 마이페이지, 프로필 수정화면 등)에서 어떻게 쓰이고 있는지**를 캡처하여 증빙해야 합니다. (활용하지 않는 정보는 API 설정에서 수집안함으로 빼야 합니다.)
